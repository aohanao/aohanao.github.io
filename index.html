项目背景与痛点

在传统的重工业设计中，有限元分析（CAE）和仿真（如 Abaqus 物理引擎）极其复杂，高度依赖资深工程师手工调试网格、调整边界荷载和物理参数。单次方案迭代往往需要花费 3 到 5 天时间，且仿真出错的定位成本极高。项目旨在利用大语言模型（LLM）的多步骤规划和反思自愈能力，构建一个“自然语言输入，自动决策与执行仿真”的智能体系统。

核心系统架构

本系统基于 LangGraph 编排有向有权状态图（Directed Acyclic Graph with States），以状态共享机制为核心，采用 Executor-Critic (执行者-裁判者) 双轨演进架构：

  • Planner 节点 (意图路由)：对用户自然语言输入进行分析，判断是“常规技术咨询(Chat/RAG)”还是“仿真执行(Simulate)”，并进行参数初置。
  • Extractor 节点 (参数提取)：从历史文档、规范以及用户对话中提取并补全结构力学参数，经过严格的物理合规校验。
  • Coder 节点 (代码渲染)：根据提取的参数自动渲染 Abaqus Python 运行脚本。
  • Executor 节点 (物理沙箱)：在受控的 Docker 仿真沙箱中拉起 Abaqus 执行仿真脚本，并捕获 stderr/stdout 运行日志。
  • Critic 节点 (物理准入/自愈)：根据执行结果和物理边界阈值判断仿真是否通过。若仿真报错或未通过，自动携带报错日志回流给 Extractor 节点触发 Reflexion 反思重试 机制。

关键技术创新

双核心记忆架构

利用滑动窗口记忆 Compressor 压缩短期上下文，减少 Token 浪费；同时将历史成功仿真参数保存至 ChromaDB 向量数据库，新会话启动时通过向量检索实现“经验闪回”，加快调优收敛。

FastMCP 工具通道

采用 Anthropic 推出的 Model Context Protocol (模型上下文协议) 规范，将外部 RAG 工具和材料规范库封装为 MCP Server，智能体通过标准化 HTTP/SSE 异步通道与之通信，实现零耦合的热插拔工具调用。

项目背景与痛点

随着多智能体协同（Multi-Agent）系统的复杂度上升，智能体的决策过程成为了一个完全的“黑盒”：智能体在哪个节点出现幻觉、调用了什么工具、耗时分布如何、以及 RAG 检索的文档到底准不准，开发和产品团队在过去只能依赖人工拉取日志排查，故障定位极其耗时，严重阻碍产品迭代。

低侵入式埋点与 Trace 系统

为解决上述黑盒痛点，平台独立设计并实现了一套零侵入式的可观测 SDK (EvalPlatformCallback)：

  • 装饰器探针埋点：使用 Python 装饰器和上下文管理器拦截智能体的核心节点，自动记录进入/退出参数、消耗 Token 及各层耗时。
  • Trace ID 贯穿管道：设计了 Trace ID 封装与传输协议，支持在多进程、多服务以及穿透 MCP 管道时保持全局一致的 Trace 追踪，实现调用流拓扑瀑布图展示。
  • 异步队列安全上报：采用独立异步线程和线程安全队列缓存储存监测数据，避免监控组件拖慢智能体核心业务流程的响应速度。

自动化评测大盘 (RAGAS + LLM-as-a-Judge)

秉承“评测即产品”的产品思维，平台集成了领先的评测引擎体系：

  • RAG 质量化量测：集成 RAGAS 框架，对智能体检索模块进行 Faithfulness（忠实度）、Answer Relevance（回答相关度）、Context Precision（检索精准度）三大维度的动态打分。
  • 决策审计 (LLM-as-a-Judge)：选用 **DeepSeek 裁判大模型** 建立裁判机制，通过设计高强度的 Prompt 模板自动审计智能体工具调用的合理性。
  • 可视化瀑布大盘：基于 Streamlit 和 Plotly 搭建瀑布图面板，展示从用户提问到最终输出的完整 Trace Span 链路，帮助开发人员进行耗时瓶颈分析与下钻诊断。

面向智慧交通基建的装备动态配置与智能调度系统

项目背景与痛点

在大型智慧公路基建（如复杂隧道、高大边坡等施工现场）中，大量的特种装备（如凿岩台车、掘进机、运输车）协同效率十分低下。由于受限于地形窄通道和物理空间约束，传统的静态规划算法无法满足动态避障和路径冲突处理的要求，导致施工窝工与成本超支。

多目标优化与路径寻优算法

项目团队研发并落地了双核心运筹算法体系：

  • 工期-成本多目标平衡最优 (NSGA-II)：融合多目标遗传算法（NSGA-II）与**熵权-TOPSIS**决策机制，在海量的配置矩阵中智能逼近帕累托前沿，自适应选择成本最低、工期最短的特种装备最优配置组合。
  • 窄通道动态避障与路径纠偏 (A* 算法重构)：针对隧道施工的窄通道时空阻隔约束，通过重构 A* 启发式代估函数并配合搜索树剪枝，实现了多设备行进冲突在线避碰，动态纠偏响应延迟控制在秒级以内。
  • 工程落地与交付：后端基于 SpringBoot 搭建调度核心，对接前端现场物联传感器和数据库，成功在真实公路施工路段完成落地部署与高可用测试。
